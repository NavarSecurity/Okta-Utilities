# okta-admin-role-reporter

Python utility for exporting Okta admin role assignments, delegated admin access, privileged users, privileged groups, custom roles, resource sets, and custom role bindings.

This utility is intended for Okta privileged access review, IAM audit preparation, delegated administration review, custom admin role documentation, migration planning, and operational evidence collection.

---

## What This Utility Does

The `okta-admin-role-reporter` utility exports Okta administrative access information from an org.

It can export:

| Export Area | Purpose |
|---|---|
| User admin role assignments | Identifies users with directly assigned Okta admin roles. |
| Group admin role assignments | Identifies groups that grant Okta admin privileges to their members. |
| Client admin role assignments | Optionally exports admin roles assigned to OAuth client/service apps when client IDs are provided. |
| Role targets | Shows scoped app or group targets for delegated admin roles. |
| Custom roles | Exports custom admin roles. |
| Custom role permissions | Exports permissions attached to custom roles. |
| Resource sets | Exports resource sets used by custom admin role bindings. |
| Resource set bindings | Shows which custom roles are bound to which resource sets. |
| Binding members | Shows users, groups, or clients assigned through resource set bindings. |
| Privileged group members | Optionally exports members of groups that have admin role assignments. |

This utility does not create, update, delete, assign, or remove admin roles. It is read-only.

---

## Safety Model

This utility is read-only.

It does not modify Okta users, groups, clients, roles, role targets, custom roles, permissions, resource sets, or bindings.

Dry-run validation:

```bash
okta-admin-role-reporter --config config.json --dry-run
```

Real export:

```bash
okta-admin-role-reporter --config config.json
```

The utility should not print or write Okta API tokens, bearer tokens, client secrets, private keys, passwords, authorization headers, or session values.

## Requirements

- Python 3.10 or newer
- Okta admin API token
- Okta org URL
- Network access to the Okta tenant

The API token should be generated by an Okta admin account with permission to read admin roles, users, groups, custom roles, resource sets, and bindings.

For OAuth-scoped access, the equivalent read scope is typically `okta.roles.read`. This utility uses SSWS token authentication by default.

## Setup

Run these commands from the utility folder.

Create a virtual environment:

```bash
python3 -m venv .venv
```

Activate the virtual environment on macOS or Linux:

```bash
source .venv/bin/activate
```

Activate the virtual environment on Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Install the utility:

```bash
python -m pip install -e .
```

Copy the environment example:

```bash
cp .env.example .env
```

Copy a sample configuration file:

```bash
cp samples/config.export.sample.json config.json
```

Open `.env` and update the Okta values.

## How to Read the Main Reports

### `privileged_assignments.csv`

Start here for audit review.

This file shows users, groups, or clients with high-risk admin roles.

Important columns:

| Column | Meaning |
|---|---|
| `principalType` | Whether the admin assignment is on a user, group, or client. |
| `principalName` | User name, group name, or client identifier. |
| `login` | User login when the principal is a user. |
| `roleType` | Okta role type such as `SUPER_ADMIN`, `ORG_ADMIN`, or `CUSTOM`. |
| `roleLabel` | Human-readable role label. |
| `assignmentSource` | Whether the role is directly assigned to a user, assigned to a group, or assigned to a client. |

### `admin_role_assignments.csv`

This is the full flat list of exported admin role assignments.

Use this when reviewing all admin access, not only high-risk access.

### `admin_role_targets.csv`

This shows role target scope for delegated administration.

Example interpretation:

```text
A group has APP_ADMIN, but only for one target application.
```

or:

```text
A user has GROUP_MEMBERSHIP_ADMIN, but only for one target group.
```

### `resource_set_bindings.csv`

This shows custom role bindings between resource sets and roles.

Use this when Okta custom admin roles are enabled.

### `binding_members.csv`

This shows who receives access through custom role/resource set bindings.

Members can be users, groups, or client apps.

## Group Role Assignment Behavior

Okta exposes a direct endpoint for users with role assignments. For groups, the utility lists groups and checks each group for assigned roles.

For large orgs, use selected group mode to reduce runtime.

Copy the selected-groups sample:

```bash
cp samples/config.selected-groups.sample.json config.json
cp samples/groups.txt input/groups.txt
```

Open `input/groups.txt` and list the groups you want to inspect.

Then run:

```bash
okta-admin-role-reporter --config config.json
```

---

## Optional Client Role Assignment Export

Client role assignment export is disabled by default.

To use it, add client IDs to `input/clients.txt`:

```bash
cp samples/clients.txt input/clients.txt
```

Open `config.json` and set:

```json
{
  "includeClientRoleAssignments": true,
  "includeClientRoleTargets": true,
  "clientSelection": {
    "mode": "ids",
    "clientIds": [],
    "clientFile": "input/clients.txt"
  }
}
```

Then run:

```bash
okta-admin-role-reporter --config config.json
```

## Troubleshooting

### 401 Unauthorized

Confirm the Okta org URL and API token in `.env`.

```bash
cat .env
```

The org URL should look like:

```text
https://your-org.okta.com
```

Do not include `/api/v1` in `OKTA_ORG_URL`.

---

### 403 Forbidden

The API token may not have permission to read admin roles, users, groups, custom roles, resource sets, or bindings.

Use an admin-generated token with sufficient read access.

---

### Group role export is slow

Large orgs can have many groups. The utility must check group role assignments per group.

Use selected group mode:

```bash
cp samples/config.selected-groups.sample.json config.json
cp samples/groups.txt input/groups.txt
okta-admin-role-reporter --config config.json
```

---

### Request failures appear but the run succeeds

If `continueOnRequestError` is true, the utility records request failures and continues.

Review:

```text
request_failures.csv
execution_report.json
```

This is useful when some endpoints are unavailable because of licensing, org type, API permissions, or feature availability.

## Security Notes

- Do not commit `.env`.
- Do not commit raw privileged access exports from client environments unless they have been reviewed.
- Treat admin role exports as sensitive access control evidence.
- Review `privileged_assignments.csv` before sharing any report.
- Validate whether group-assigned roles create effective admin access through group membership.
- Use least privilege for API tokens where possible.

---

## Practical Use Cases

Use this utility to:

- Identify all directly assigned Okta admins.
- Identify groups that grant admin privileges.
- Review Super Admin and Org Admin assignments.
- Review delegated App Admin or Group Admin scope.
- Export custom admin role permissions.
- Document resource set bindings for custom admin roles.
- Prepare access review evidence for SOX, ISO, HIPAA, SOC 2, or internal audit.
- Compare privileged access before and after migration or cleanup.
- Support least-privilege review of Okta administration.

---

## Notes

Admin role assignments are high-risk security data. Review generated files before storing or sharing them.

For production environments, run the report from a controlled workstation and store output in an approved evidence location.
